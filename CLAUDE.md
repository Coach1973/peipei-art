# 佩佩老師個人品牌頁｜給這台電腦的 Claude CLI 工作說明

> 這份檔案會在你（Claude CLI）打開這個資料夾工作時自動讀到，不需要額外指令。
> 這是陳佩君（佩佩）老師的個人品牌行銷頁專案，只有這一個網站，跟教練其他系統無關，也不需要碰教練其他系統。

## 這是什麼

單頁網站，只有一個 `index.html` 檔案，所有文字、版面都寫在裡面。照片放在 `images/` 資料夾。

測試站網址（GitHub Pages，push 之後約 30 秒～1 分鐘自動更新）：
**https://coach1973.github.io/peipei-art/**

## 🔴 重要邊界（請務必遵守）

- 這個專案目前只做到「GitHub 測試站」這一步。**真正推上正式對外網站（Netlify）不是這台電腦的工作**，那一步永遠由大樹教練親口說「推」，並且在教練自己的環境執行。
- 這台電腦的 Claude CLI **只負責改這個 repo、push 到 GitHub**，不要嘗試部署到 Netlify、不要新增/修改其他網站或系統。
- 不確定的地方（例如版面要不要大改、文案方向）先問清楚再動手，不要自己猜。

## 常見指令怎麼處理

**「我要換掉這張照片」**
教練/佩佩老師會給一張新照片。存進 `images/` 資料夾，可以直接覆蓋同檔名，或用新檔名並修改 `index.html` 裡對應的 `<img src="...">` 路徑。換完務必用瀏覽器打開 `index.html` 確認顯示正常（照片方向、比例有沒有跑掉）。

**「我要修改這張圖」**
先確認對方要的是「換掉照片本體」還是「調整這張圖在頁面上的顯示方式（大小/位置/裁切）」，不確定就先問清楚再動手。

**「我要新增這個字」／「我要刪掉這個字」**
直接在 `index.html` 對應段落做文字增刪，只改文字內容，不要動到 HTML 標籤結構本身。改完用瀏覽器打開確認排版沒有跑掉。

## 每次改完之後的固定流程

1. 用瀏覽器打開本機 `index.html`（或 `python3 -m http.server` 開本機預覽）確認畫面正常
2. `git add -A && git commit -m "說明這次改了什麼"`
3. `git push`
4. 回報時附上：改了什麼、GitHub 測試站網址、commit 的短 hash

## 快速開始（每次開工先做）

```powershell
cd <這個 repo 在這台電腦上的資料夾路徑>
git pull
```
確認是最新版本之後，就可以直接下達「我要換掉這張照片」這類指令。
